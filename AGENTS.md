# AGENTS.md

This repository uses GSD with Codex.

## Core rules

- Do not push directly to main.
- Do not implement anything during project intake.
- Do not write application code until the GSD planning files are created and approved.
- Ask one essential question at a time during project intake.
- Keep the MVP small and avoid overengineering.
- Do not add dependencies without asking first.
- Do not change architecture unless explicitly approved.
- Do not refactor unrelated files.
- Work on one phase at a time.
- Execute only the approved GSD phase.
- Keep changes small and reviewable.

## GSD workflow

1. Start with GSD project intake.
2. Create planning files:
   - .planning/PROJECT.md
   - .planning/REQUIREMENTS.md
   - .planning/ROADMAP.md
   - .planning/STATE.md
3. Stop and ask for roadmap approval.
4. Discuss one phase.
5. Plan one phase.
6. Execute one phase only.
7. Verify work.
8. Prepare a PR.

## Reporting

At the end of every task, summarize:

- What changed
- Files changed
- What was not completed
- Tests/checks run
- Any risks, assumptions, or questions
