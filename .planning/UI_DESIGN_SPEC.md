# UI Design Spec

## Direction

The Personal Learning Reader should feel like a premium private study room: calm, focused, content-first, and professional.

It should feel like a beautiful personal knowledge reader, not a dashboard, file manager, SaaS admin panel, or children's app.

## Visual Style

- Warm off-white page background.
- Charcoal primary text.
- Pale, low-contrast borders.
- One restrained accent such as deep sage or ink blue.
- Minimal shadows, if any.
- No large gradients, glass effects, or decorative clutter.

## Layout Principles

- Keep the library focused on choosing a lesson to read.
- Keep the Reading View focused on the simplified lesson.
- Avoid nested cards and dashboard widgets.
- Use generous spacing around reading content.
- Let source references feel secondary and calm.

## Typography

- Use the existing system font stack unless a font change is separately approved.
- Body text should be comfortable for long reading.
- Section headings should be clear but not oversized.
- Metadata should be small, muted, and easy to scan.

## Components

- Lesson cards should feel like editorial reading entries.
- Status labels should be subtle: quiet green/sage for `ready`, muted warm neutral for `draft`.
- Suggested visual sections should look like calm learning notes, not generated artwork.
- Source references should look like reference notes, not file-manager attachments.

## Mobile Behavior

- iPhone layout should use one column.
- Reading text should stay large enough to read comfortably.
- Side padding should feel generous without wasting space.
- Source references should appear below the lesson body on mobile.

## Avoid

- Dashboard metrics.
- File-manager layouts.
- Corporate SaaS styling.
- Childish illustrations or playful UI.
- Heavy shadows.
- Dense sidebars.
- Loud badges.
- Tiny reading text.
- Full-width long article lines.
